Shell Script File Extensions: .sh vs .bash

Both .sh and .bash files are shell scripts, i.e., programs written for a command-line shell.

Shell POSIX Bash Scripting

.sh files

  • Usually contain POSIX-compliant shell script.
  • Can be run in any POSIX shell (e.g., sh, bash, dash, ksh, zsh).
  • More portable across Unix-like systems.

Example:

#!/bin/sh
echo "Hello from a POSIX shell script!"

.bash files

  • Specifically contain Bash scripts (Bourne Again SHell).
  • May use Bash-only features (arrays, extended globbing, [[ ]] tests, etc.).
  • Should be run with bash explicitly.

Example:

#!/bin/bash
for i in {1..5}; do
    echo "Hello from Bash: $i"
done

Main difference

Aspect .sh .bash
Language POSIX shell (portable) Bash-specific (may not work in sh)
Portability High Lower (requires Bash)
Typical Shebang #!/bin/sh #!/bin/bash

In practice

  • Use .sh for maximum compatibility.
  • Use .bash when you need Bash-specific features.
